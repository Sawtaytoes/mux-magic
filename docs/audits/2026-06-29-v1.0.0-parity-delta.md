# 2026-06-29 — v1.0.0 feature-parity audit: DELTA

A second, independent parity pass against `server-v1.0.0` (`ff92625b`, last commit with the legacy `packages/web/public/builder/` tree) vs current `feat/mux-magic-revamp`. Motivated by the user hitting real regressions (NSF) and suspecting worker 60's audit ([v1.0.0-feature-parity.md](v1.0.0-feature-parity.md)) under-counted.

**Finding:** worker 60 marked nearly everything "Ported" without field-/affordance-level verification and filed only workers 7a/7b. **Command field-level parity is actually clean** — every v1→current field difference is an intentional rename (`yamlCodec.ts` round-trip maps) or a deliberate server change. The real losses are all in the **lookup / results / modal UX layer**, which worker 60 never inspected. 17 new regressions found below; none were previously tracked.

> Status legend: **missing** (gone), **regressed** (degraded), **partial** (works in some cases). Severity is user-impact. None of these is filed in MANIFEST.md or the prior audit.
