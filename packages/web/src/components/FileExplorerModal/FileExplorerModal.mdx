# FileExplorerModal

A full-screen file browser modal with two modes: **browse** (default — view and delete files) and **picker** (when `pickerOnSelect` is set — confirm-folder mode used by `PathVariableCard`'s Browse flow). Clicking a video file row opens the standalone `VideoPreviewModal` (worker 58) — the player no longer lives inside this component, so playback works from any caller (e.g. `PromptModal`) regardless of whether the explorer is open.

Sortable by name, duration, size, or modification time. Directories always sort first regardless of column. The delete-mode badge shows whether deletes go to the OS Recycle Bin or are permanent — derived from `/files/delete-mode` per current path (some paths can't trash, e.g., network shares).

## Behaviour

- **Open** → `fileExplorerAtom` set with `{ path, pickerOnSelect }`. `path` becomes the initial `currentPath`; `pickerOnSelect` decides browse vs picker mode.
- **Navigate** → click a directory row or any breadcrumb segment. Both reset selection and re-fetch listing + delete-mode.
- **Sort** → click column header. Same-column click flips direction; different column resets to ascending.
- **Select files** → row checkbox (directories disabled). Header checkbox toggles all *files* (not directories — they can't be deleted from this UI).
- **Delete** → `Delete selected` posts `DELETE /files` with the array of paths. Failure list is shown in a window.alert.
- **Picker confirm** → `📌 Use this folder` calls `pickerOnSelect(currentPath)`, closes the modal, and forwards the result to whoever opened it (typically `PathVariableCard`).
- **Video playback** → clicking a video file row sets `videoPreviewModalAtom` — `VideoPreviewModal` (mounted at the app top level by `BuilderPage`) renders the `FileVideoPlayer` overlay. Audio-codec probing and the transcode fallback live in `FileVideoPlayer` itself.
- **Esc** → closes the explorer. The video preview owns its own Escape handler (z-[60] overlay, sits above this modal).

## Atom wiring

| Atom | Role |
|------|------|
| `fileExplorerAtom` | Visibility + initial path + picker callback. `null` ⇒ closed |
| `videoPreviewModalAtom` | Set when the user clicks a video file row — opens the standalone preview overlay |

## Server endpoints

| Endpoint | Purpose |
|----------|---------|
| `GET /files/list?path=&includeDuration=1` | Directory listing (with video duration probing) |
| `GET /files/delete-mode?path=` | Reports `"trash"` vs `"permanent"` for the current path |
| `DELETE /files` body `{ paths }` | Bulk delete |
