# PageHeader

The top bar of the builder page. Hosts navigation (New Sequence, Jobs link), undo/redo, collapse-all / expand-all, dry-run + failure-mode toggles, the run buttons, and YAML import/export icons.

All actions are wired directly to Jotai atoms and hooks — there is no legacy bridge layer.

## Behaviour

- **Responsive menus** → Two collapsible panels (`page-menu-nav` and `page-menu-controls`) that show/hide based on `openMenu`. Toggle buttons (`#page-nav-toggle`, `#page-controls-toggle`) flip the state; clicks outside the panels close them via a document `mousedown` listener.
- **Esc** → closes open modal (YamlModal / SequenceRunModal via atom) and any open responsive panel.
- **Dry-run badge** → Renders only when `dryRunAtom` is true. Clicking it (or the toggle in the controls menu) flips the atom AND updates `?fake` in the URL (`history.replaceState`).
- **Failure-mode toggle** → Only rendered when dry-run is on. Updates `?fake=failure` / `?fake=success` in the URL. State is per-tab and survives reload via the URL.
- **Undo/redo enable state** → Driven by `canUndoAtom` / `canRedoAtom` from `historyAtoms.ts`; updated by `useBuilderActions`.
- **Run buttons** → `▶ Run Sequence` calls `runSequence` (client-side: the browser drives each step in order via `/commands/:name`); `▶ Run via API` calls `runViaApi` (server-side: one umbrella `/sequences/run` job that honors parallel groups + named-output chaining). Both are disabled while `runningAtom` is true.

## Atom wiring

| Atom | Role |
|------|------|
| `dryRunAtom` | Drives the badge + toggle. Backed by `?fake` URL param |
| `failureModeAtom` | "Simulate Failures" toggle. Backed by `?fake=failure` URL param |
| `runningAtom` | Disables run buttons while a sequence is in flight |
| `canUndoAtom` / `canRedoAtom` | Enable/disable undo + redo buttons |

## Actions (via useBuilderActions)

| Action | Role |
|--------|------|
| `startNew` | "New Sequence" button |
| `undo` / `redo` | Toolbar buttons |
| `setAllCollapsed(bool)` | Collapse-all / expand-all |
| `runSequence` | ▶ Run Sequence button (client-side, serial) |
| `runViaApi` | ▶ Run via API button (server-side umbrella job) |
| `addPath` | "Add Path" button |
| `copyYaml` | Copy YAML icon |
